# Game Design Document — Badoz Incremental
_Version 0.1 — 2026-06-15_

---

## Concept

**Tagline:** From a lonely dev with a laptop to the world's most famous solopreneur — as fast as possible.

You play as an indie developer trying to build a SaaS product, grow its ARR, and retire rich. The game is an **incremental with a hard win condition**: reach the target ARR and hit "Retire." 

What makes it different from typical idle games: it is **designed to be speedrun**. The first playthrough is a discovery experience where powerful mechanics unlock progressively. Subsequent runs start fresh but the *player* carries full knowledge, enabling tighter strategies and faster completions. A timer is always visible. Finished runs are recorded on a personal dashboard.

---

## Theme & Narrative

You are a developer. You open your laptop. You start writing code.

The world is the modern SaaS ecosystem: shipping features, acquiring users, fighting churn, posting on social media, and progressively delegating everything to AI models so you can focus on strategy. No employees — just you, your product, and an ever-growing fleet of AI agents.

The tone is dry and self-aware: upgrade names, flavor text, and milestone messages should wink at real solopreneur culture (indie hackers, "build in public", MRR screenshots, etc.).

---

## Win Condition

**Retire at $5M ARR.**

When the player reaches $5M ARR, a "Retire" button appears. Clicking it ends the run, records the elapsed time, and displays the player on their personal speedrun dashboard.

The target feels large and intimidating on the first run. By run 3, a skilled player should be able to hit it in a fraction of the original time.

---

## Core Resource Loop

```
Code → Features → Customers → MRR → ARR
                        ↑
               Reputation (passive acquisition)
```

**Code** is the primary manual resource. Writing code (clicking or auto-generating) produces features.

**Features** drive two things:
- Customer acquisition (more/better features attract users)
- Customer retention (fewer features = higher churn)

**Customers** pay a monthly subscription. Their count × their tier determines MRR.

**MRR** (Monthly Recurring Revenue) compounds into **ARR** over time. ARR is the game's primary progress bar.

**Reputation** is a passive multiplier earned through marketing actions (blog posts, X/Twitter threads, launch events). It boosts acquisition rate without requiring direct action.

## Mechanics

### Early game — The Grind
| Mechanic | Description |
|---|---|
| **Write Code** | Manual click. Generates raw code units. |
| **Ship Feature** | Spend code to add a feature to your product. Increases acquisition rate and reduces churn. |
| **Set Price** | Choose your subscription price tier (higher = more MRR per user, lower = easier to acquire). |
| **Post on X** | Spend time to gain Reputation. Passive acquisition multiplier. |
| **Cold outreach** | One-time acquisition burst. Diminishing returns per use. |

### Mid game — Delegation begins
Unlocked progressively as ARR milestones are reached.

| Mechanic | Description |
|---|---|
| **Hire AI Coder** | AI model that generates code automatically at a fixed rate. Each has a cost/quality tradeoff. |
| **AI Support Agent** | Reduces churn rate passively. |
| **AI Marketer** | Generates Reputation passively, improves acquisition. |
| **Pricing experiments** | A/B test price tiers to find the optimal conversion rate. |
| **Launch on Product Hunt** | One-time massive acquisition event. Reputation-gated. |

### Late game — Scaling
| Mechanic | Description |
|---|---|
| **AI Product Manager** | Autonomously decides which features to ship for maximum retention. |
| **Growth loops** | Referral program: each customer brings fractional new customers. |
| **Enterprise tier** | Unlock a higher-value customer segment with lower churn. |
| **Press coverage** | Large one-time Reputation spike, triggers acquisition surge. |
| **AI CEO mode** | Everything is automated. The player becomes purely strategic. |

## Progression Milestones

These act as "aha moments" on the first run, naturally pacing the discovery of new mechanics.

| ARR Milestone | Unlock |
|---|---|
| $1K | First AI Coder becomes available |
| $5K | AI Support Agent unlocked — player first understands churn as a lever |
| $10K | Product Hunt launch becomes available |
| $25K | Pricing experiments unlocked |
| $50K | AI Marketer unlocked |
| $100K | Enterprise tier unlocked |
| $500K | AI Product Manager unlocked |
| $1M | Growth loops + Press coverage unlocked |
| $5M | **"Retire" button appears** |

## Speedrun Meta

### How it works
- Each run starts from zero: no resources, no AI, no reputation.
- The **player** retains all knowledge. No in-game carry-over bonuses (no prestige currency, no permanent upgrades).
- The game's depth comes from *routing*: what do you unlock first? When do you post vs. when do you code? When do you raise your price?

### The Dashboard
After each completed run, the player sees:
- Run number
- Completion time
- Final ARR at retirement
- A brief "build log" (key milestones and when they were hit)

The dashboard persists across all runs and becomes a personal speedrun history.

### Intended run arc
| Run | Experience |
|---|---|
| Run 1 | Discovery. ~60–90 min. Player doesn't know what unlocks or when. |
| Run 2 | Experimentation. Player tests optimal unlock order. ~30–40 min. |
| Run 3+ | Execution. Tightly routed. Sub-20 min possible for experts. |

## Open Design Questions

- Should there be any in-game carry-over mechanic (e.g., a "playbook" bonus from previous run), or keep it pure knowledge-only?
- What is the game clock? Real time or an accelerated in-game time (days/months of company growth)?
- Should churn be visible as a counter, or felt indirectly through stagnating MRR?
- Should the player be able to name their SaaS product?
- Leaderboard: local only, or eventually shareable (online)?
