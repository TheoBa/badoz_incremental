## Concept

**Tagline:** From a lonely dev with a laptop to the world's most famous solopreneur — as fast as possible.

You play as an indie developer trying to build a SaaS product, accumulate wealth, and retire as the richest solopreneur in history. The game is an **incremental with a hard win condition**: accumulate $1B in total money and hit "Retire."

What makes it different from typical idle games: it is **designed to be speedrun**. The first playthrough is a discovery experience where powerful mechanics unlock progressively. Subsequent runs start fresh but the *player* carries full knowledge, enabling tighter strategies and faster completions. A timer is always visible. Finished runs are recorded on a personal dashboard.

---

## Theme & Narrative

You are a developer. You open your laptop. You start writing code.

The world is the modern SaaS ecosystem: shipping features, acquiring users, fighting churn, posting on social media, and progressively delegating everything to AI models so you can focus on strategy. No employees — just you, your product, and an ever-growing fleet of AI agents.

The tone is dry and self-aware: upgrade names, flavor text, and milestone messages should wink at real solopreneur culture (indie hackers, "build in public", MRR screenshots, etc.).

## Run Start — Name Your Product

The very first action of every run is naming the SaaS product. A single text input is shown before the game loop begins. The name is purely cosmetic but appears everywhere throughout the run (mission flavor text, MRR displays, milestone messages). It creates attachment and makes each run feel personal. No name = run doesn't start.

---

## Win Condition

**Retire at $1,000,000,000 (total money).**

Money is earned through freelance missions and SaaS subscriptions. It accumulates in the player's wallet. Spending money on investments or Frontier Lab subscriptions accelerates growth but delays the win — creating the core tension of every run.

When the player reaches $1B, a "Retire" button appears. Clicking it ends the run, records the elapsed time, and posts the result to the personal speedrun dashboard.

## Time System

The game runs on an **accelerated in-game clock** driven by a single configurable constant:

```
TICK_RATE = 1  // real seconds per in-game hour (adjust freely during iteration)
```

With the default value of 1:
- **1 real second = 1 in-game hour**
- **1 in-game day = 24 real seconds**
- **1 in-game month = ~12 real minutes** (30 days × 24s)
- **A 90-minute run ≈ 225 in-game days ≈ 7.5 in-game months**

`TICK_RATE` is the single variable to tune if the pacing feels too fast or too slow. Everything in the engine derives from it.

Time units and what they govern:
- **In-game hour** — base tick. RCUs generated per tick. RCU/h and MB/h stats update every tick.
- **In-game day** (24 ticks) — customer acquisition resolves. Freelance missions refresh. Frontier Lab subscriptions charged upfront. Post on X cooldown resets.
- **In-game month** (720 ticks) — each customer's subscription renews or churns. At sufficient customer volume, daily acquisition makes churn feel like a smooth continuous flow.

RCUs are spent by the player at any time — there is no time gate on spending, only on generation.

**Subscription billing rule:** Frontier Lab plan costs are deducted at the start of each in-game day. Changing a plan takes effect at the start of the *next* day — the current day's plan and cost are locked in at day start. Downgrading mid-day does not refund the difference.

## Core Resource Loop

```
[Click / AI Coder] → RCUs / hour
              ├──→ Freelance Missions tab  → 💰 Money (direct, early game)
              └──→ Ship Feature tab        → SaaS Product properties ↑
                                                    │
                                         Customers × Subscription Price
                                                    │
                                            Monthly 💰 Money (MRR)
                                                    │
                              ┌─────────────────────┴──────────────────────┐
                       Investment tab                             Frontier Lab plans
                    (time-limited boosts)                      (daily burn, daily boost)
                                                                            │
                                                                      $1B → Retire
```

### Resources

**RCUs (Raw Code Units)** — the primary craftable resource. Generated per in-game hour by clicking "Write Code" or passively by the AI Coder agent. Spent in Freelance Missions and Ship Feature (and to unlock/upgrade agents).

**Money** — the win resource. Earned from freelance missions and SaaS MRR. Spent on Investments and Frontier Lab subscriptions. Core tension: spend for compounding growth vs. accumulate toward $1B.

**Reputation** — a passive multiplier built up over time through Post on X and certain investments. Compounds permanently within a run. Boosts the SaaS product's Marketing Stream.

## The SaaS Product

The product is the player's core passive income engine. It has three properties displayed as raw numbers (rounded to 3 significant digits):

| Property | Effect |
|---|---|
| **Customer Satisfaction** | Multiplier on conversion rate: how likely a potential client becomes a paying customer each day. |
| **Customer Retention** | Multiplier on monthly renewal rate: how likely a customer stays after their subscription period ends. |
| **Marketing Stream** | Number of potential clients that "visit" the product each in-game day. |

The product also has a **Subscription Price** — not fixed, but not freely adjustable either. It starts at a base value and can only be raised by the player at specific milestone-gated moments. Raising the price is a deliberate, player-triggered action with real trade-offs.

**Price increase mechanics:**
- Only available when a **lifetime money earned** milestone unlocks a "Pricing Round" (`Price_Round_T1`, `Price_Round_T2`, …). Lifetime money earned is a monotonically increasing counter — it never decreases when the player spends — so it's a reliable proxy for overall run progress.
- Player chooses when to activate the raise; the option doesn't expire, so timing is a skill
- Raising price applies immediate negative pressure to **Customer Satisfaction** and **Customer Retention** (demand responds negatively to a price hike)
- Despite the dip, the higher price per customer should produce a net revenue increase if the timing is right — more revenue per seat outweighs the temporary churn bump
- Raises are one-way: price can never be lowered

This makes pricing a tension point: raise too early and you spike churn before your retention is strong enough to absorb it; raise too late and you leave revenue on the table. On later speedruns, knowing the optimal raise timing is a meaningful skill expression.

Each in-game day and month, the game resolves:
```
New customers / day  = Marketing Stream × conversion_rate(Customer Satisfaction)
Churned customers / month = existing customers × churn_rate(Customer Retention)
Monthly revenue      = active customers × Subscription Price
```

## Tabs

### Tab 1 — Write Code

The home base. A "Write Code" button generates RCUs per click. Once the AI Coder is active, passive RCU generation begins and this tab becomes less about clicking and more about monitoring.

Always displayed in this tab:
- Current RCU balance
- **RCU/h** — current hourly RCU generation rate
- **MB/h** — Money Burned per hour (sum of all active Frontier Lab daily costs, converted to an hourly rate)

---

### Tab 2 — Ship Feature

Spend RCUs to permanently improve the SaaS product's three properties (Customer Satisfaction, Customer Retention, Marketing Stream). Features have increasing RCU costs on a standard incremental curve.

This tab is the long-game investment: no direct money output, but massive compounding effect on MRR over time.

---

### Tab 3 — Freelance Missions

Three missions available per in-game day. Each shows:
- Flavor name (e.g. "Fix a WordPress plugin", "Ship an auth flow")
- RCU cost
- Money reward

Missions refresh at day start. Completing a mission spends the RCUs and adds the money instantly.

#### Developer tiers

Mission quality scales with **total lifetime RCUs generated** — a proxy for coding experience, not wealth:

| Tier | Unlock (total RCUs) | Flavor | RCU cost | Payout |
|---|---|---|---|---|
| **Junior** | 0 | Bug fixes, landing pages, WordPress | Low | Low |
| **Senior** | TBD | Features, API integrations, SDKs | Medium | Medium |
| **Lead** | TBD | Systems design, codebase audits, solo sprints | High | High |
| **10x** | TBD | MVP in a weekend, rescue a dying startup | Very high | Very high |

The three daily missions are always drawn from the player's current highest tier.

#### Rush option

A **Rush** modifier can be applied to any mission: spend extra RCUs on top of the base cost to double the money reward.

Unlocked via a **one-time RCU payment** of `Rush_RCU_Cost` (= `Freelance_RCU_T1` — the same threshold that unlocks Senior missions). This means Rush becomes affordable naturally just as the player levels up to Senior, without requiring a separate grind. Once paid, Rush is permanently available for the rest of the run. The option is always shown on each mission card but greyed out if the player lacks sufficient RCUs to afford both the base cost and the Rush surcharge. This keeps the decision visible without blocking the base mission.

---

### Tab 4 — Investment

Spend money on time-limited boosts and one-off growth events.

| Investment | Effect | Duration |
|---|---|---|
| **Cold Outreach campaign** | Temporary spike in Marketing Stream | 1 in-game day |
| **SEO push** | Gradual Marketing Stream boost | 7 in-game days |
| **Sponsored newsletter** | Flat Reputation gain | Instant |
| **Launch on Product Hunt** | Massive one-time Marketing Stream + Reputation event | Instant (once per run) |
| **Press coverage** | Large Reputation spike | Instant (limited uses) |

More investments unlock at money milestones.

---

### Tab 5 — AI Agents / The Frontier Lab

Unlocked at the $100 money milestone. This tab is styled distinctly from the rest of the UI — darker, slightly ominous — to reflect the Faustian nature of the vendor.

#### About The Frontier Lab

**The Frontier Lab** is a shadowy AI company with suspiciously capable models and suspiciously aggressive pricing. Their tagline: *"We align with your wallet."* All agent flavor text, plan names, and upgrade descriptions should carry this dry, slightly sinister tone.

#### Agent lifecycle

**1. Unlock** — pay a one-time RCU cost to access an agent. Agents are listed in The Frontier Lab's catalogue in unlock order, each more expensive than the last. Locked agents are visible but greyed out, with their RCU cost shown.

**2. Subscribe** — once unlocked, the agent starts on Free (idle). The player scrolls through plan tiers to set an active subscription. The selected plan's cost is charged at the start of each in-game day. The current day's plan cannot be changed retroactively.

**3. Upgrade model** — separately from the plan, spend RCUs to upgrade the agent's underlying model version (e.g. v1 → v2 → v3). Each version upgrade multiplies the effect at every plan tier.

#### Subscription tiers

| Plan | Daily cost | Multiplier |
|---|---|---|
| **Free** | $0 | — (idle) |
| **Hobbyist** | $5 | 1.5× |
| **Growth** | $25 | 4× |
| **Scale** | $100 | 12× |
| **Infernal** | $500 | 40× |

```
effective_boost = plan_multiplier × model_version_multiplier
```

#### Core agents

| Agent | Unlock cost | Boosts |
|---|---|---|
| **AI Coder** | Low | RCU/h generation |
| **AI Support** | Medium | Customer Retention (churn reduction) |
| **AI Marketer** | Medium-high | Marketing Stream + Reputation/day |

#### Later agents (milestone-gated)

| Agent | Unlock gate | Boosts |
|---|---|---|
| **Growth Hacker** | TBD Lab spend | Marketing Stream conversion rate |
| **PR Bot** | TBD Lab spend | Passive press events (Reputation spikes) |
| **Product Manager** | TBD Lab spend | Auto-ships Ship Feature upgrades |

#### Strategic tension

Frontier Lab subscriptions drain money every day. Near $1B, downgrading all agents to Free and coasting on accumulated MRR is often the optimal finish. Knowing *when* to pull the plug is a core speedrun skill. The MB/h display in Tab 1 keeps this cost visible at all times.

---

### Tab 6 — Post on X

Available from day 1. Once per in-game day, the player can post a thread. Each post grants a small permanent Reputation boost that compounds with all previous posts. No streak bonus — each post is independent.

The mechanic is intentionally easy to forget. It provides a meaningful advantage over a full run to players who remember it daily, but it is never a must-do and never telegraphed as critical. Only the most attentive players will post every single day.

Cooldown resets at in-game day start.

## KPI Dashboard

A persistent dashboard panel (always accessible, not a tab) displays a small set of graphs and live stats. No ROI indicators per agent — the player is expected to reason about efficiency themselves.

### Live stats (always visible in Tab 1)
- **RCU/h** — current hourly RCU generation
- **MB/h** — Money Burned per hour (Frontier Lab daily costs ÷ 24)

### Dashboard graphs
- **Lifetime money** — cumulative money earned over the run (not current balance)
- **Lifetime RCUs** — cumulative RCUs generated
- **MRR over time** — how monthly recurring revenue has evolved
- **Daily cash burn** — Frontier Lab costs per day over time

All values rounded to 3 significant digits in display.

## Early Game Mechanic Flow

1. Player enters product name → run starts
2. Player clicks **Write Code** → RCU/h shown
3. Player opens **Freelance Missions** → first money earned
4. Player opens **Ship Feature** → product properties begin rising, small MRR appears
5. **$100 reached → AI Agents tab unlocks**; player unlocks AI Coder with RCUs → RCU/h goes passive
6. Player now splits RCUs: Freelance (cash), Ship Feature (MRR), Agent upgrades (efficiency)
7. Player sets AI Coder to Hobbyist → MB/h shown for the first time → first burn awareness
8. AI Support and AI Marketer unlock → churn drops, Marketing Stream grows
9. First Investment available → spend-vs-save tension begins
10. Player shifts from clicking to routing and plan management

## Balancing Variables

All numeric thresholds that require playtesting use named variables. This keeps the GDD readable, makes couplings explicit, and means the code only needs to update one constant file to retune.

```
// Time
TICK_RATE           = 1          // real seconds per in-game hour

// Freelance tier unlock thresholds (total lifetime RCUs)
Freelance_RCU_T1    = TBD        // unlocks Senior tier
Freelance_RCU_T2    = TBD        // unlocks Lead tier
Freelance_RCU_T3    = TBD        // unlocks 10x tier

// Frontier Lab cumulative spend thresholds
Lab_Money_T1        = TBD        // unlocks AI Support
Lab_Money_T2        = TBD        // unlocks AI Marketer
Lab_Money_T3        = TBD        // unlocks Product Hunt (Investment tab)
Lab_Money_T4        = TBD        // unlocks Growth Hacker agent
Lab_Money_T5        = TBD        // unlocks Enterprise tier
Lab_Money_T6        = TBD        // unlocks PR Bot agent
Lab_Money_T7        = TBD        // unlocks Product Manager agent
Lab_Money_T8        = TBD        // unlocks Growth loops
Lab_Money_T9        = TBD        // unlocks AI CEO mode

// Lifetime money earned thresholds (trigger Pricing Round availability)
Price_Round_T1      = TBD        // first pricing round available
Price_Round_T2      = TBD        // second pricing round (if any)

// One-time unlocks
Rush_RCU_Cost       = Freelance_RCU_T1   // Rush unlock costs the same as Senior tier threshold
```

Exact values for all TBD variables to be set during the implementation and playtesting phase.

## Progression Milestones

Three separate tracks gate different unlocks.

**RCU milestones** (total lifetime RCUs generated):

| Total RCUs | Unlock |
|---|---|
| 50 | AI Coder available (Agents tab) |
| `Freelance_RCU_T1` | Freelance tier: **Senior** |
| `Freelance_RCU_T2` | Freelance tier: **Lead** |
| `Freelance_RCU_T3` | Freelance tier: **10x** |
| `Rush_RCU_Cost` (= `Freelance_RCU_T1`) | Rush option permanently unlocked |

**Total money milestones** (bootstrapping only):

| Total money | Unlock |
|---|---|
| $100 | **AI Agents tab unlocks** |

**Frontier Lab spend milestones** (cumulative money burned on agent subscriptions):

Spending on the Lab is the primary mid-to-late game progression lever. The more you invest in AI, the more powerful tools become available — making Lab commitment mandatory rather than optional.

| Cumulative Lab spend | Unlock |
|---|---|
| `Lab_Money_T1` | AI Support available |
| `Lab_Money_T2` | AI Marketer available |
| `Lab_Money_T3` | Product Hunt launch (Investment tab) |
| `Lab_Money_T4` | Growth Hacker agent available |
| `Lab_Money_T5` | Enterprise subscription tier unlocked |
| `Lab_Money_T6` | PR Bot agent available |
| `Lab_Money_T7` | Product Manager agent available |
| `Lab_Money_T8` | Growth loops unlocked |
| `Lab_Money_T9` | AI CEO mode — strategic endgame |
| $1B total money | **"Retire" button appears** *(win condition — not a Lab spend gate)* |

Exact thresholds to be set during balancing, calibrated against plan costs and intended run length. A player on Hobbyist ($5/day) should hit early milestones naturally within a first run; reaching late milestones requires committing to higher plans.

## Speedrun Meta

### How it works
- Each run starts from zero: no RCUs, no money, no agents, no reputation.
- The **player** retains all knowledge. No in-game carry-over bonuses.
- Depth comes from *routing*: when to pivot from freelance to SaaS-first? When to upgrade agents vs. ship features? When to throttle Frontier Lab to stop the burn?

### The Speedrun Dashboard
After each completed run, the player sees:
- Run number
- Completion time
- Split times: first $1K / $10K / $100K / $1M / $1B
- Money source breakdown (freelance % vs. MRR %)

The dashboard persists across all runs. Online leaderboard planned for a future release.

### Intended run arc
| Run | Experience |
|---|---|
| Run 1 | Discovery. ~60–90 min. Player explores organically. |
| Run 2 | Experimentation. Tests routing (freelance-first vs. SaaS-first). ~30–40 min. |
| Run 3+ | Execution. Optimal route, precise agent throttling. Sub-20 min for experts. |
